# 16.2 ‚Äî Build the curated home dashboard at `/`

## Scope

Replace the stub `/` page (from 16.1) with the curated home layout per features/16 ¬ß"Layout Specification". Net effect: `/` no longer looks like `/markets`. The user gets a focused dashboard with the most live activity (open + pending resolution), capped at 6 + 3 cards, no category filter, no closed/resolved sections.

`/markets` is unchanged.

## Acceptance criteria

### Page content

- [ ] **No `<h1>` reading "Markets"** on `/`. The hero / featured-market component (if used) can have its own heading; the page itself doesn't.
- [ ] **No `CategoryFilter` component** rendered on `/`. The filter is `/markets`-only.
- [ ] **Hero / featured-market section** at top. Reuse the existing component (see `frontend/src/components/hottest-market.tsx` or whatever the existing page already uses for its top section). One market.
- [ ] **"Open Markets" section** with `markets.filter(m => m.status === "open").slice(0, 6)` cards.
- [ ] **"Pending Resolution" section** with `markets.filter(m => m.status === "pending_resolution").slice(0, 3)` cards.
- [ ] **No "Closed" or "Resolved" sections.** Those live exclusively at `/markets`.
- [ ] **"Browse all markets ‚Üí"** link or button at the bottom, navigating to `/markets`. Visible on every variant (signed-in or anon).

### Implementation

- [ ] New client component `frontend/src/app/home-client.tsx` (or similar) holds the layout. Server component at `frontend/src/app/page.tsx` prefetches `marketsQuery()` and renders `<HomeClient />`.
- [ ] Reuses `MarketCard` and `marketsQuery` ‚Äî no new backend endpoint, no new TanStack Query.
- [ ] Same access spec as today (matrix ¬ß5 `/` row): `render-readonly` for `anon`/`demo`, `render` for signed-in roles. The page itself doesn't need explicit gating beyond what `MarketCard` already does for betting controls.

### Empty states

- [ ] If there are zero open markets, render a friendly empty-state ("No open markets right now ‚Äî be the first to propose one!" with a CTA to `/markets/create`).
- [ ] If there are zero pending-resolution markets, hide the section entirely (don't render an empty header).

### Tests (in the same PR if Playwright is set up; otherwise queued for 15.2)

- [ ] Add `frontend/e2e/page-gating/home.spec.ts`:
  - `/` does NOT have an `<h1>` containing "Markets".
  - `/` does NOT have a `[data-testid="category-filter"]` (or whatever selector identifies the filter ‚Äî confirm).
  - `/` shows ‚â§6 open-market cards and ‚â§3 pending-resolution cards.
  - `/` has a "Browse all markets" link to `/markets`.
  - `/markets` retains its `<h1>` "Markets" + category filter + all sections.

## Notes

- **Empty-state copy** ‚Äî pick text that matches the app's voice. "No open markets right now ‚Äî be the first to propose one!" is one example; tweak to taste.
- **Featured-market selection** ‚Äî reuse whatever logic the existing markets page uses for its hero / "hottest market" today. If there isn't a clear pick, just use the first open market sorted by created_at desc.
- **Category filter side-effect** ‚Äî make sure the filter still works at `/markets`. The split should not have moved any state-management code.
- **Performance** ‚Äî the same `marketsQuery()` hit serves both `/` and `/markets`; TanStack Query caches it. Visiting `/` then `/markets` shouldn't double-fetch.
- After this lands, doc 15.2 page tests (if already done) need updating to match the new layout difference. If 15 hasn't shipped yet, add the home-page assertions to 15.2's acceptance criteria as a follow-up note.
