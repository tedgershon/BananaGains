# 13.9 — Gap B-3: relocate admin-resolve route under `/api/admin/...`

## Scope

Close matrix Gap B-3 by moving the dead `markets.py::admin_resolve` handler (lines 470–496) to `admin.py::admin_resolve_market` and updating the path to `POST /api/admin/markets/{id}/resolve`. Replace the inline `is_admin` check with `Depends(require_admin)`. **No frontend changes** because no frontend caller exists today (`rg "markets/admin"` returns no matches outside specs).

The relocation is a prerequisite for the future Gap B-4 work (`/admin/admin-review` UI), which will call this relocated route. Doing the move now keeps the security/maintenance surface clean even before the UI exists.

## Acceptance criteria

- [ ] **Delete** `markets.py::admin_resolve` (lines 470–496) entirely. No replacement in `markets.py`.
- [ ] **Add** `admin.py::admin_resolve_market` with the same business logic:
  - Path: `POST /api/admin/markets/{id}/resolve` (note the path lives at `/api/admin/markets/{id}/resolve` because the admin router prefix is `/api/admin`).
  - Dependencies: `Depends(require_admin)` (replacing the deleted inline `current_user.get("is_admin")` check).
  - Behavior: identical to the deleted handler — accepts a `resolved_outcome` body (YES / NO / AMBIGUOUS), updates `markets.status = 'resolved'`, sets `resolved_outcome`, triggers payout/refund flows, raises `AppError(400, ErrorCode.MARKET_NOT_ADMIN_REVIEW, ...)` if the market is not in `admin_review` status.
  - All exception handling uses `AppError` per the conventions established in 13.1 / 13.4.
- [ ] **Frontend:** no changes needed (no caller exists today). Run `rg "/api/markets/admin/" frontend/src` to confirm zero matches before merging.

## Verification

```bash
# Old path is gone
curl -i -X POST http://localhost:8001/api/markets/admin/<some-uuid>/resolve \
  -H "Authorization: Bearer <admin token>" -d '{"resolved_outcome": "YES"}'
# expected: 404 (route no longer exists)

# New path works for admin
curl -i -X POST http://localhost:8001/api/admin/markets/<admin_review-uuid>/resolve \
  -H "Authorization: Bearer <admin token>" -d '{"resolved_outcome": "YES"}'
# expected: 200, market status now 'resolved'

# New path rejects user
curl -i -X POST http://localhost:8001/api/admin/markets/<id>/resolve \
  -H "Authorization: Bearer <user token>" -d '{"resolved_outcome": "YES"}'
# expected: 403, body {"detail":{"code":"ADMIN_REQUIRED","message":"Admin access required."}}

# New path rejects anonymous
curl -i -X POST http://localhost:8001/api/admin/markets/<id>/resolve \
  -d '{"resolved_outcome": "YES"}'
# expected: 401

# Frontend has no callers
rg "/api/markets/admin/" frontend/src
# expected: zero matches
```

## Notes

- This route currently has **no frontend caller**. If during your audit you discover one (e.g., a hardcoded `fetch` in a component), update the call site in this PR.
- The future `/admin/admin-review` page (Gap B-4, separate doc) will be the first frontend caller. That work is **not** in scope here — this PR's job is just the relocation.
- If the team decides to instead delete the `admin_review` terminal state from the state machine (the alternative option recorded in matrix §8 B-4), this relocated route also gets deleted later. For now, relocate per spec.
- Do **not** keep both old and new paths "for compatibility" — there's no caller, so there's nothing to be compatible with.
