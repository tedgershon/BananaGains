# 13.1 — `ErrorCode` enum, `AppError` exception, global handlers

## Scope

Add the canonical error contract to `backend/observability.py` (the `ErrorCode` enum and the `AppError` exception class) and wire two global handlers in `backend/main.py` (validation and last-resort unhandled). **No router changes** in this PR — those start in 13.3.

After this PR merges, every existing `HTTPException(...)` continues to work (since `AppError` is a subclass), and the unhandled-exception handler ensures any forgotten case returns the envelope shape rather than leaking a stack trace.

## Acceptance criteria

### `backend/observability.py`

- [ ] Adds `ErrorCode(str, Enum)` with **all** entries from features/extension/13-backend-hardening.md §1b. Members are `str`-valued so `code.value` produces the JSON string directly.
- [ ] Adds `AppError(HTTPException)` matching the signature in features/extension/13-backend-hardening.md §1c:
  ```python
  AppError(
      status_code: int,
      code: ErrorCode,
      message: str,
      *,
      original: BaseException | None = None,
      extras: dict[str, Any] | None = None,
      errors: list[dict[str, Any]] | None = None,
  )
  ```
- [ ] When `original is not None`, `AppError.__init__` opens a Sentry `push_scope`, sets the `error_code` tag to `code.value`, attaches every `extras` key/value as a Sentry extra, and calls `sentry_sdk.capture_exception(original)`. Then it constructs the FastAPI `HTTPException` with `detail={"code": code.value, "message": message}` and `errors` appended if provided.
- [ ] Imports `sentry_sdk` directly (this is the only place outside the SDK init that touches `sentry_sdk`).

### `backend/main.py`

- [ ] Adds `@app.exception_handler(RequestValidationError)` `validation_handler` that returns:
  ```json
  {"detail": {"code": "VALIDATION_FAILED", "message": "Validation failed.", "errors": [{"field": "...", "message": "...", "type": "..."}]}}
  ```
  Status 422. The `errors` array is built from `exc.errors()`; `field` joins `loc` segments with `.`.
- [ ] Adds `@app.exception_handler(Exception)` `unhandled_exception_handler` that:
  - Re-raises if `isinstance(exc, HTTPException)` (lets FastAPI's native handler take over for `AppError` and any remaining direct `HTTPException` use).
  - Otherwise calls `report_and_safe_message(exc, "Internal server error.")` (Sentry capture) and returns the envelope with status 500 and `code: INTERNAL_ERROR`.

### Smoke testing

- [ ] **Temporary** smoke check: raise `AppError(409, ErrorCode.BET_INSUFFICIENT_BALANCE, "Test", original=Exception("smoke test"))` from the `/` health check, hit it, confirm the envelope shape and Sentry event. **Revert before merging.**
- [ ] Existing tests pass: `cd backend && pytest tests/`.

## Verification

```bash
# Validation envelope
curl -i -X POST http://localhost:8001/api/markets \
  -H "Authorization: Bearer <user token>" -H "Content-Type: application/json" -d '{}'
# expected: 422 with body {"detail":{"code":"VALIDATION_FAILED","message":"Validation failed.","errors":[...]}}

# Unhandled-exception envelope (after temp smoke endpoint that raises ValueError):
curl -i http://localhost:8001/__force-error
# expected: 500 with body {"detail":{"code":"INTERNAL_ERROR","message":"Internal server error."}}

# AppError envelope (during smoke test in / health check):
curl -i http://localhost:8001/
# expected: 409 with body {"detail":{"code":"BET_INSUFFICIENT_BALANCE","message":"Test"}}
# Sentry event with error_code=BET_INSUFFICIENT_BALANCE tag
```

## Notes

- Do **not** migrate any router in this PR. The point is that the new envelope and handlers exist, ready for routers to use in 13.3–13.5.
- The `unhandled_exception_handler`'s `isinstance(exc, HTTPException)` guard is critical — without it, every legitimate `AppError` would be re-captured and double-counted.
- If you need to add a new `ErrorCode` not in §1b, update the enum in this PR, not later.
