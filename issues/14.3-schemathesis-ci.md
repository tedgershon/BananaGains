# 14.3 — Schemathesis: property-based + unauth fuzzing (optional CI gate)

## Scope

Add property-based contract tests via `schemathesis`. Loads the FastAPI app's OpenAPI spec **in-memory** (no network) and generates randomized requests within the schema's constraints. Two scenarios:

1. **Schema conformance:** for any valid input, the response status is one of the documented options and the body matches the declared `response_model`. Catches drift between code and `response_model`.
2. **Unauth fuzzing:** the same generated requests, but with `Authorization` header stripped. Every endpoint not in `PUBLIC_ENDPOINTS` must return 401 or 403. Catches "I added an endpoint and forgot to gate it."

**Optional and CI-flagged.** Set `SCHEMATHESIS=1` to run; defaults to skipped so daily PR feedback isn't blocked by occasional flaky generated edge cases.

## Acceptance criteria

### Dependency

- [ ] `backend/requirements-dev.txt` adds `schemathesis>=3.30.0` (or current stable).

### `backend/tests/test_schemathesis.py` (new)

- [ ] Schema loaded via `schemathesis.openapi.from_asgi("/openapi.json", main.app)` (in-process, no HTTP).
- [ ] All tests in this file marked with `@pytest.mark.skipif(os.environ.get("SCHEMATHESIS") != "1", reason="schemathesis is opt-in via SCHEMATHESIS=1")`.
- [ ] **Conformance test:**
  ```python
  @schema.parametrize()
  def test_api_does_not_crash(case):
      response = case.call_asgi()
      case.validate_response(response)
  ```
- [ ] **Unauth fuzzing test:**
  ```python
  PUBLIC_ENDPOINTS = {
      ("GET", "/api/markets"),
      ("GET", "/api/markets/{id}"),
      ("GET", "/api/markets/hot"),
      # ... mirror matrix §7 cells where `anon` is 200/201/204
  }

  @schema.parametrize()
  def test_unauth_endpoints_correctly_gated(case):
      case.headers.pop("Authorization", None)
      response = case.call_asgi()

      if response.status_code in (401, 403, 404, 422):
          return
      if response.status_code in range(200, 300):
          assert (case.method, case.path.template) in PUBLIC_ENDPOINTS, (
              f"Unauth request to {case.method} {case.path.template} succeeded "
              f"but is not in PUBLIC_ENDPOINTS — possible unauth bypass"
          )
  ```
- [ ] `PUBLIC_ENDPOINTS` is hand-maintained; document this in a comment with a pointer to the matrix §7 cells it mirrors.

## Notes

- **`from_asgi` does not require `/openapi.json` to be exposed in production.** Schemathesis grabs the spec directly from the ASGI app object via Python imports. Once doc 14.4 disables `/openapi.json` in production, schemathesis tests still work locally and in CI because they don't go through HTTP.
- **Iteration count** — schemathesis defaults can over-generate (slow CI). Tune via `@schema.parametrize(max_examples=20)` if runtime is too long.
- **Flakiness handling** — if a generated payload hits a flaky edge case (e.g., timezone parsing), file an issue, narrow the schema, or add a `@schema.given` filter to exclude that input shape. Don't disable the test wholesale.
- **PRs that add new endpoints** must also update `PUBLIC_ENDPOINTS` if the endpoint is intentionally anonymous. The unauth-fuzzing test will catch this on the first CI run (or first local run with `SCHEMATHESIS=1`).
- This is the soft safety net. The hard contract is doc 14.2.
