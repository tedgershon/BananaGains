# 12.3 — Backend: tag Sentry context with the authenticated user UUID

## Scope

Two-line edit in `backend/dependencies.py`: after `_resolve_user` returns a profile in `get_current_user` and `get_current_user_optional`, call `set_user_context(profile["id"])` from `observability`. Result: every authenticated request automatically carries the user's UUID in Sentry events and structured log records.

This is small enough that it could have been bundled with 12.2, but kept separate so 12.2's diff stays purely about infrastructure (no behavior changes per request).

## Acceptance criteria

- [ ] `backend/dependencies.py` imports `set_user_context` from `observability`.
- [ ] `get_current_user`: after `profile = _resolve_user(credentials.credentials, supabase)`, add `set_user_context(profile["id"])` before `return profile`.
- [ ] `get_current_user_optional`: same addition inside the `try` block, before `return profile`. The except branch (which catches `HTTPException` and returns `None`) does **not** call `set_user_context` — anonymous requests keep `_user_id` as `None`.
- [ ] `**Send `id` (UUID), never `andrew_id`.**` Per design doc rationale.
- [ ] No other functional changes.

## Verification

```bash
# Sign in as any user, hit any authenticated endpoint
curl -X GET http://localhost:8001/api/auth/me \
  -H "Authorization: Bearer <token>"

# In Sentry (after intentionally raising an error from any authenticated path),
# the captured event's `user.id` should equal the Supabase profile UUID,
# NOT andrew_id and NOT email.
```

For an anonymous request to a `get_current_user_optional` endpoint, the captured event's `user` should be unset (not the previous request's user — that's what `clear_user_context` in the middleware ensures).

## Notes

- This PR is the sentinel proof that 12.2's `ContextVar` plumbing works under real auth flows.
- If `Sentry.setUser` is called in `set_user_context` and then a different user makes a request on the same Python process (e.g., serverless container reuse), the `RequestContextMiddleware` `finally` block from 12.2 clears the context — confirm this in testing by alternating two test users and checking Sentry for cross-contamination.
