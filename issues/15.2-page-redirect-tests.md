# 15.2 — Page redirect tests (matrix §5)

## Scope

For every row in AUTHZ_MATRIX §5, write a Playwright test per role asserting:

- `render` cells: page renders, expected element visible.
- `render-readonly` cells: page renders, but mutating UI controls are absent (delegated to 15.3 for the deep checks; 15.2 just confirms the page itself loads).
- `redirect→/auth` cells: page redirects to `/auth`, **no flash** of protected content.
- `redirect→/` cells: page redirects to `/`, no flash.
- `404` cells: HTTP 404 returned.

The "no flash" guarantee is the regression-blocker. Server-side redirects pass; client-side `useEffect` redirects fail.

  - `frontend/e2e/page-gating/admin.spec.ts` (`/admin`, `/admin/review`, `/admin/admin-review`*, `/admin/stats`, `/admin/users`)

### Per page

- [ ] One `test.describe(<route>)` block. Inside, one `test()` per role from matrix §5. Tests cover all five roles (`anon`, `demo`, `user`, `admin`, `super_admin`) — match the cells exactly. **N tests per route, ~5 roles × ~14 routes ≈ 70 tests.**
- [ ] **No-flash assertion** for redirects:
  ```ts
  test("anon → redirect→/auth", async ({ page }) => {
    const responsePromise = page.waitForResponse((r) => r.url().includes("/markets/create"));
    await page.goto("/markets/create");
    const response = await responsePromise;
    // Server-side redirect: response status is 307 with Location: /auth, body never reaches the browser
    expect([307, 302, 308]).toContain(response.status());
    await expect(page).toHaveURL(/\/auth/);
    // Even if the browser briefly hits /markets/create, there should be NO marker element from the form
    await expect(page.getByLabel("Title")).not.toBeVisible();
  });
  ```
- [ ] **Render assertion** uses a marker element specific to the page (e.g., the `<h1>` of "Markets" for `/markets`). Don't assert against generic shells.

## Notes

- **Pages with rendering states** (e.g., loading spinner) need waiting for. Use `page.waitForLoadState("networkidle")` after `goto` to avoid race conditions.
- **The `demo` role** — recall the matrix says demo is a frontend illusion (no real backend role). For the Playwright test, `signInAs("demo")` should set up the demo sentinel UUID in `localStorage` per `SessionProvider`'s logic, then assert behavior matches the cell.
- **`render-readonly` cells** in this PR only assert the page renders — leave the deep "no mutating controls visible" check to 15.3 to avoid duplicating the §6 sweep.
- After 15.7 (Gap B-1 fix from doc 13), the `/markets/create` anon test should pass without `test.fixme`. Verify in this PR's setup.
- The matrix is the spec; if a test fails, fix the code, not the spec. If the matrix itself is wrong, update it in a separate PR.
