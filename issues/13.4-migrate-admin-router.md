# 13.4 ŌĆö Migrate `backend/routers/admin.py` exceptions to `AppError`

## Scope

Same pattern as 13.3, applied to `backend/routers/admin.py`. Smaller surface, two known leak sites (lines 188, 300). Auth dependencies (`require_admin`, `require_super_admin`) are already correct per the audit; no changes needed there.

This PR does **not** add the relocated `admin_resolve_market` route ŌĆö that's part of 13.9.

## Acceptance criteria

- [ ] Every `raise HTTPException(...)` in `admin.py` is replaced with `raise AppError(...)` using the `ErrorCode` enum.
- [ ] The two known `f"...: {e}"` interpolations at lines 188 and 300 are gone. Underlying exceptions captured via `original=exc`.
- [ ] Per matrix ┬¦7e: `PUT /api/admin/users/{id}/role` returning "cannot change own role" maps to `AppError(400, ErrorCode.ADMIN_CANNOT_CHANGE_OWN_ROLE, ...)`.
- [ ] Per matrix ┬¦7e: any "admin required" / "super-admin required" path uses `ErrorCode.ADMIN_REQUIRED` / `ErrorCode.SUPER_ADMIN_REQUIRED` (status 403). Note these are usually emitted by `require_admin` / `require_super_admin` themselves; if those dependencies still raise plain `HTTPException`, update them in `backend/dependencies.py` to raise `AppError` so the envelope shape is consistent. **Document any `dependencies.py` change in the PR description.**
- [ ] No behavioral changes ŌĆö every endpoint continues to respond with the same status code for the same input. Just a different exception type and body shape.
- [ ] Existing tests pass.

## Mapping table

| Old behavior | ErrorCode | Status |
|---|---|---|
| Admin required (from `require_admin`) | `ADMIN_REQUIRED` | 403 |
| Super-admin required (from `require_super_admin`) | `SUPER_ADMIN_REQUIRED` | 403 |
| Cannot change own role | `ADMIN_CANNOT_CHANGE_OWN_ROLE` | 400 |
| Backroll on resolved market | `MARKET_ALREADY_RESOLVED` | 409 |
| Review on non-pending market | `MARKET_NOT_PENDING_REVIEW` | 400 |
| Generic 5xx wrapping | `INTERNAL_ERROR` | 500 |
