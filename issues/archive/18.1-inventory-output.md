- No `seeds/` or `backend/supabase/seeds/` directory exists today; the
  single seed-script row is `seed-plan.md` at repo root.

### Recommended 18.2 split

**Single PR.** Total of 24 findings, of which the genuine source edits
are:

- 7 frontend files modified to import constants from a new
  `frontend/src/lib/constants.ts` (create the new file; update both
  page.tsx duplicates and possibly relocate `OPTION_COLORS`).
- 1 frontend file modified to add a `// DEMO MODE: ...` comment block
  in `SessionProvider.tsx`.
- 1 frontend file modified to replace an inlined sentinel literal
  with `useSession()` (`markets/[id]/detail-client.tsx`).
- 1 file move (`seed-plan.md` → `seeds/PLAN.md`) plus a new
  `seeds/README.md`.
- 1 docstring trim in `backend/dependencies.py`.

That's ~10 files touched in 18.2 — well within a single reviewable PR.
**Recommendation: do not split into 18.2a/18.2b.** If 18.2 grows during
implementation (e.g., `OPTION_COLORS` relocation surfaces more
consumers), the split can be made then. The natural split-point, if
needed, is "frontend constants consolidation" vs "demo-scaffolding
docs + seed-plan move + backend docstring."

### Coordination note for the 18.2 PR description

State explicitly in 18.2's PR description whether the cleanup happens
**before** or **after** doc 17 ships, since both PRs touch the
`MAX_MARKET_*_LENGTH` constants. The cleaner ordering is:

1. **17 ships first** with the constants still inline at their current
   locations; 18.2 then moves the (possibly updated) values to
   `lib/constants.ts` and de-duplicates.

— or, equivalently —

2. **18.2 ships first**, moving the constants to `lib/constants.ts`
   at their current values; 17 then edits the values in their new
   location.

Either ordering works; the PR that ships second avoids a merge
conflict by following the other's lead.
