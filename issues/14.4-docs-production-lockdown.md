# 14.4 — Disable `/docs`, `/redoc`, `/openapi.json` in production

## Scope

FastAPI exposes interactive API docs and the raw OpenAPI schema on three default routes: `/docs`, `/redoc`, `/openapi.json`. Useful for local dev and tests; in production they're a discovery surface for anyone curious about the backend's shape.

This PR disables all three when the app is running in production.

## Acceptance criteria

### `backend/main.py`

- [ ] At app construction, conditionally pass `docs_url=None`, `redoc_url=None`, `openapi_url=None` based on `settings.sentry_environment`:
  ```python
  is_production = settings.sentry_environment == "production"
  app = FastAPI(
      title="BananaGains API",
      version="0.1.0",
      docs_url=None if is_production else "/docs",
      redoc_url=None if is_production else "/redoc",
      openapi_url=None if is_production else "/openapi.json",
  )
  ```
- [ ] No other behavioral changes.

### Verification (must pass before merging)

- [ ] In dev (`SENTRY_ENVIRONMENT=development`): all three routes return 200.
- [ ] In production-like environment (`SENTRY_ENVIRONMENT=production`): all three return 404.
- [ ] Schemathesis tests (doc 14.3) still pass when set up — they use `from_asgi`, which loads the schema in-memory regardless of HTTP exposure.

## Verification

```bash
cd backend

# dev
SENTRY_ENVIRONMENT=development uv run uvicorn main:app --port 8001 &
curl -s -o /dev/null -w "%{http_code}\n" http://localhost:8001/docs
# expected: 200
curl -s -o /dev/null -w "%{http_code}\n" http://localhost:8001/openapi.json
# expected: 200
kill %1

# production-like
SENTRY_ENVIRONMENT=production uv run uvicorn main:app --port 8001 &
curl -s -o /dev/null -w "%{http_code}\n" http://localhost:8001/docs
# expected: 404
curl -s -o /dev/null -w "%{http_code}\n" http://localhost:8001/redoc
# expected: 404
curl -s -o /dev/null -w "%{http_code}\n" http://localhost:8001/openapi.json
# expected: 404
kill %1
```

## Notes

- This is a **defense-in-depth** measure, not a security boundary. Anyone with the source can derive the same schema by reading `backend/routers/`. The point is to not actively *advertise* the schema to drive-by traffic.
- **Local development unaffected.** `/docs` is great for exploring the API — only production hides it.
- **Tests unaffected.** Schemathesis loads the schema via `from_asgi`, which doesn't go through HTTP. Other test fixtures use `TestClient`, which also bypasses HTTP routing. Both work fine when `openapi_url=None` in the live app.
- This issue can ship at any time — it doesn't depend on the test infrastructure work in 14.1–14.3. Listed under doc 14 only because it's the same hardening theme.
