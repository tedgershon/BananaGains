# 15.1 — Playwright install + config + fixtures

## Scope

Stand up the Playwright e2e infrastructure and the four custom fixtures that doc 15 builds on. **First Playwright tests in the codebase.** No matrix-driven assertions yet — this PR is pure scaffolding plus one smoke test per fixture proving they work.

## Acceptance criteria

### Install

- [ ] `pnpm add -D @playwright/test` — adds to `frontend/devDependencies`.
- [ ] `pnpm exec playwright install chromium` — installs the browser binary.
- [ ] `frontend/package.json` adds scripts:
  ```json
  "e2e": "playwright test",
  "e2e:ui": "playwright test --ui",
  "e2e:debug": "playwright test --debug"
  ```

### `frontend/playwright.config.ts`

- [ ] `baseURL` reads from `process.env.PLAYWRIGHT_BASE_URL ?? "http://localhost:3000"`.
- [ ] One project (`chromium`) to start. Firefox/Webkit deferred.
- [ ] `trace: "on-first-retry"`, `screenshot: "only-on-failure"`, `video: "retain-on-failure"`.
- [ ] `retries: process.env.CI ? 1 : 0`.
- [ ] `webServer` config (optional) that boots `pnpm dev` if `PLAYWRIGHT_BASE_URL` isn't reachable. Skip in CI (services brought up explicitly there).

### `frontend/e2e/fixtures.ts`

- [ ] `signInAs(role: "user" | "admin" | "super_admin", opts?: { creatorOf?: string })`:
  - Mints a JWT with the test secret + role's UUID (mirrors doc 14.1's test profiles).
  - Writes to `localStorage` keys that Supabase reads (e.g., `sb-<project-ref>-auth-token`).
  - Does NOT navigate the page; caller is responsible for `page.goto(...)`.
  - The `opts.creatorOf` parameter doesn't affect the JWT — it's a marker for the test's logical setup.
- [ ] `signOut()`:
  - Clears auth-related `localStorage` and cookies.
- [ ] `seedMarket(opts: { status?, creatorRole?, ...fields })`:
  - Calls a backend test endpoint (or operates on the test DB directly) to insert a market row.
  - Returns `{ id, ... }`.
  - **Cleanup:** records the inserted ID; suite teardown deletes them.
- [ ] `seedBet(market, opts: { role, side, amount?: 10 })`:
  - Inserts a bet for the given user on the given market with the given side.
  - Returns the bet ID.
- [ ] `setBalance(role: "user" | "admin", amount: number)`:
  - Direct UPDATE on the test profile's balance. Used for `BET_INSUFFICIENT_BALANCE` toast tests.

### `frontend/e2e/smoke/fixtures.spec.ts` (new)

- [ ] Four trivial tests proving each fixture works in isolation:
  - `signInAs("user")` then `page.goto("/")` shows the signed-in nav.
  - `seedMarket({status: "open"})` returns a valid `{id}` and the market appears at `/markets/{id}`.
  - `seedBet(market, {role: "user", side: "yes", amount: 5})` reduces the test user's balance by 5.
  - `setBalance("user", 0)` then `signInAs("user")`, navigate to a market, balance shows 0.

### CI

- [ ] GitHub Actions job `e2e` (in addition to existing `pytest` and `lint`):
  - Sets up node, installs frontend deps, installs Playwright browsers (use cache to avoid 100MB downloads on every run).
  - Boots backend (`uvicorn main:app`) with test DB env vars in the background.
  - Boots frontend (`pnpm dev` or `pnpm build && pnpm start`).
  - Runs `pnpm e2e --reporter=html`.
  - Uploads `playwright-report/` as artifact on failure.

### Documentation

- [ ] `frontend/README.md` adds a short "Running e2e tests" section pointing to `pnpm e2e` and `pnpm e2e:ui` for local use.

## Notes

- **JWT minting** is the trickiest fixture. The simplest option: backend exposes a dev-only endpoint `POST /api/__test/mint-token` (admin-only, gated on `SENTRY_ENVIRONMENT == "development"`) that takes a UUID and returns a signed JWT. Playwright fixture calls this endpoint at session start. Document the trade-off in the PR description.
- **DB cleanup discipline** — every fixture that inserts must record the IDs and clean them up at suite teardown. Use Playwright's `test.afterEach` or worker-scoped fixtures.
- **Don't gate this PR on the matrix tests existing.** This is foundation; 15.2/15.3/15.4 build on it.
- **Browser binary cache** — Playwright browser downloads can take minutes in CI. Use the `actions/cache` action with key `playwright-${{ hashFiles('package-lock.json') }}` to skip re-downloads.
