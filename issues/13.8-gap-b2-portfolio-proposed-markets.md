# 13.8 — Gap B-2: new `/api/portfolio/proposed-markets` endpoint, replace leak source on `/portfolio`

## Scope

Close matrix Gap B-2: `/portfolio` currently shows other users' proposed and denied markets because it queries `GET /api/markets?status=pending_review|denied` — endpoints that do not filter by `creator_id`. This is a real cross-user data leak.

**Fix:** introduce a dedicated `GET /api/portfolio/proposed-markets` endpoint that filters by `current_user["id"]`, and replace the two leaky `marketsQuery` calls in `frontend/src/app/portfolio/page.tsx` with a new `proposedMarketsQuery()`.

## Acceptance criteria

### Backend

- [ ] `backend/routers/portfolio.py` adds `get_my_proposed_markets`:
  ```python
  @router.get("/portfolio/proposed-markets", response_model=list[MarketResponse])
  async def get_my_proposed_markets(
      current_user: dict = Depends(get_current_user),
      supabase: Client = Depends(get_supabase_client),
      token: str | None = Depends(get_user_token),
  ):
      """Markets the current user has proposed (pending_review or denied), newest first."""
      with user_auth(supabase, token):
          result = (
              supabase.table("markets")
              .select("*")
              .eq("creator_id", current_user["id"])
              .in_("status", ["pending_review", "denied"])
              .order("created_at", desc=True)
              .execute()
          )
      return result.data or []
  ```
- [ ] No other endpoint changes.
- [ ] Migration of any existing `HTTPException` in this file already happened in 13.5 — this PR just adds the new route.

## Notes

- **Why a new endpoint and not filter the existing one:** see design doc §3b. tl;dr — `GET /api/markets?status=pending_review` is also used by admin tooling; role-conditional filtering would create a behavior fork that's hard to test, and a purpose-built endpoint makes the access control obvious from the route.
- Add the `STALE.PORTFOLIO` constant if it doesn't exist yet (look in `frontend/src/lib/query/staleTimes.ts`); align with the existing portfolio query staleness so the two refresh in lockstep.
- Do **not** modify the existing `marketsQuery({status: "pending_review"})` call sites in admin code — those are correct (admins legitimately need cross-user views).
