# 16.1 — Route split: separate `/` and `/markets` page implementations

## Scope

Pure refactor. Today `frontend/src/app/page.tsx` is `export { default } from "./markets/page"` — root and `/markets` render the identical thing. This PR makes them two separate page implementations that *initially serve identical content*, so 16.2 can diverge `/` cleanly without an entangled diff.

**No user-visible change.** Both routes render exactly what `/markets` renders today, after this PR. The visible split happens in 16.2.

## Acceptance criteria

- [ ] `frontend/src/app/page.tsx` no longer re-exports from `markets/page`. Instead, it has its own implementation that **for now** does the same `qc.prefetchQuery(marketsQuery())` + `<MarketsClient />` pattern. ~10 lines.
- [ ] `frontend/src/app/markets/page.tsx` and `markets-client.tsx` work unchanged when accessed via `/markets`. Verify by visiting both URLs — both render the existing markets browse identically.
- [ ] Update `frontend/src/components/navbar.tsx` `NAV_LINKS`:
  - Change the existing `{ href: "/", label: "Markets" }` entry to `{ href: "/markets", label: "Markets" }`.
  - **No new "Home" entry.** The brand element (line 28, `<Link href="/">`) already serves as the home link — keep it as is.
- [ ] No other behavioral changes.
