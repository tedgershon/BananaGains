# 18.1 — Inventory: enumerate hardcoded data + classify

## Scope

A read-mostly PR that produces a complete, classified inventory of hardcoded non-persisted data in the repo. Output is a structured markdown table (added inline to the PR description and committed to `issues/18.1-inventory-output.md` for permanence).

**No source-file changes** in this PR — except adding the inventory output file.

## Acceptance criteria

### Sweep

- [ ] Run each of the following `rg` queries from repo root. Paste raw output (or a summary if voluminous) into the inventory output:
  ```bash
  rg "DEMO_USER_ID|isDemo|demo_user|sample[_A-Z]?[Dd]ata|MOCK|mock[_A-Z]?[Dd]ata|hardcod" --no-heading -n frontend/src backend
  rg "^\s*(//|#)\s*(TODO|FIXME|XXX|HACK)\b" --no-heading -n frontend/src backend
  rg "00000000-0000-0000-0000-000000000000" --no-heading -n .
  rg "^export const [A-Z_]+\s*=\s*\[" --no-heading -n frontend/src
  rg "^[A-Z_]+\s*=\s*\[" --no-heading -n backend
  ```
- [ ] Manually inspect the top-of-file constants in:
  - `frontend/src/app/markets/create/page.tsx` (CATEGORIES, TIME_ZONE_OPTIONS, MAX_MARKET_* constants)
  - `frontend/src/app/admin/review/page.tsx` (CATEGORIES, MAX_MARKET_* constants)
  - `frontend/src/components/probability-chart.tsx` (OPTION_COLORS)
  - Any other top-of-file constant blocks the sweeps surface.

### Classify

For each finding, assign exactly one of the four categories from features/18 §"What 'dummy data' means here":

- **runtime-config** (keep — possibly relocate)
- **seed-script** (keep but isolate to `seeds/` or `backend/supabase/seeds/`)
- **demo-scaffolding** (decide: keep + document, or remove)
- **dead-code** (remove)
