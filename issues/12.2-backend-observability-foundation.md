# 12.2 — Backend: `observability.py`, request-context middleware, `main.py` wiring

## Scope

Implement the backend Sentry foundation: SDK install, the `observability` module, the request-context middleware, the global wiring in `main.py`, and the new settings in `config.py`. No router changes. Smoke-tested via a temporary debug endpoint, which is removed before merge.

This is the single largest backend PR in the observability phase. After it lands, 12.3 (a 2-line follow-up in `dependencies.py`) wires authenticated users into Sentry context.

## Acceptance criteria

### Dependencies & settings

- [ ] `backend/requirements.txt` adds `sentry-sdk[fastapi]>=2.18.0` and `python-json-logger>=2.0.7`.
- [ ] `backend/config.py` `Settings` class adds: `sentry_dsn: str = ""`, `sentry_environment: str = "development"`, `sentry_release: str | None = None`, `sentry_traces_sample_rate: float = 0.1`.

### New module: `backend/observability.py`

- [ ] Defines two private `ContextVar`s: `_request_id` and `_user_id`.
- [ ] `init_observability() -> None` — initializes Sentry SDK with `FastApiIntegration(transaction_style="endpoint")`, `StarletteIntegration()`, `LoggingIntegration(level=INFO, event_level=ERROR)`. Reads `SENTRY_DSN` from settings; **no-ops cleanly when DSN is unset** (logs a single INFO line confirming the no-op).
- [ ] `init_observability` also calls `_configure_logging()`: switches root logger to `JsonFormatter` (with `rename_fields={"levelname": "level", "asctime": "timestamp"}`) in production, plain `logging.Formatter` in development. The handler wraps the formatter with a filter class that injects `request_id` + `user_id` from the `ContextVar`s into every log record.
- [ ] `_before_send(event, hint)` strips `Authorization`, `Cookie`, `X-API-Key` headers, and `andrew_id` from `event.extra`.
- [ ] `_before_send_transaction(event, hint)` drops health-check transactions (`/`, `/health`, `/api/health`).
- [ ] `begin_request(request_id: str | None = None) -> str` — sets `_request_id` in the contextvar and returns the id (generates a UUID hex if none passed).
- [ ] `set_user_context(user_id: str | None) -> None` — sets `_user_id` and `sentry_sdk.set_user({"id": user_id})`. Used by 12.3 — no callers in this PR.
- [ ] `clear_user_context() -> None` — clears both.
- [ ] `report_and_safe_message(exc, user_message, *, level="error", extras=None) -> str` — captures `exc` to Sentry inside a `push_scope` (with `extras` attached and `scope.level = level`), returns `user_message` unchanged. **This is doc 13's foundational helper; ensure the signature matches doc 12 §"Module Shape" exactly.**

### New middleware: `backend/middleware/request_context.py`

- [ ] `backend/middleware/__init__.py` exists (empty package marker).
- [ ] `RequestContextMiddleware` extends `BaseHTTPMiddleware`. `dispatch` reads `x-request-id` from request headers (or generates one), calls `begin_request(rid)`, awaits `call_next`, sets `response.headers["x-request-id"] = rid`, and in `finally` calls `clear_user_context()`.

### Wiring: `backend/main.py`

- [ ] `init_observability()` is called **before** `FastAPI(...)` instantiation.
- [ ] `RequestContextMiddleware` is added **outermost** (added before `CORSMiddleware`) so it wraps CORS and all routers.
- [ ] No other behavioral changes — existing router includes unchanged.

### Smoke testing

- [ ] **Temporary** debug endpoint `POST /api/__sentry-test` (admin-only via `Depends(require_admin)`) raises a known exception. Used to verify Sentry capture works end-to-end. **Must be removed before merging this PR**, OR feature-flagged behind `settings.sentry_environment == "development"` if kept.

### Local dev parity

- [ ] With `SENTRY_DSN` unset: app starts, logs `"Sentry DSN not set; skipping Sentry init"` once at INFO, no events sent, no network calls. Existing tests pass.
- [ ] With `SENTRY_DSN` set to a development DSN from 12.1: hitting the `__sentry-test` endpoint produces an event in the `bananagains-backend` project with `environment:development`.

## Verification

```bash
# baseline: app boots without Sentry
cd backend
SENTRY_DSN= uv run uvicorn main:app --port 8001
# expected log line: "Sentry DSN not set; skipping Sentry init"

# with Sentry: confirm capture
SENTRY_DSN=<dev DSN from 12.1> SENTRY_ENVIRONMENT=development \
  uv run uvicorn main:app --port 8001
# trigger the temp endpoint as an admin user; check Sentry within 30s

# verify request_id round-trip
curl -i -X GET http://localhost:8001/api/leaderboard
# response includes 'x-request-id: <hex>' header
# stdout log line includes the same request_id
```

In the captured Sentry event, confirm:
- `Authorization` header is `[Filtered]`
- `andrew_id` is absent or `[Filtered]`
- `request_id` is in `extra`

## Notes

- **Do not** import `sentry_sdk` from any router or test file in this PR. The only import surface is `backend/observability.py`. Future PRs will import the helpers from there.
- Logging integration captures INFO records as breadcrumbs and ERROR+ as events. If this turns out to be too noisy in production, raise `event_level` to `CRITICAL` later.
- The `__sentry-test` endpoint must be admin-only; do not let an anonymous request trigger arbitrary exception capture.
